Graficos vectoriales

Son muy utiles ya que tienen un tamaño pequeño y son responsivos, se adaptan al cambiar el tamaño de su entorno y no se pixelan

¿Que son los graficos vectoriales?

Existen dos tipos de imagenes:

  • Imagenes ráster: Estan compuestas por una cuadrícula de pixeles, contiene información que muestra exactamente dónde se colocará cada pixel y exactamente de que color debe ser. Los más populares incluyen un mapa de bits (.bmp), PNG (.png), JPEG(jpg), GIF (.gif).
  • Imagenes vectoriales: Estan compuestas por algoritmos, código (se pueden programar manualmente) para que la computadora determine como se debe representar la imagen. Su formato es el SVG.

La diferencia se ve cuando se amplian los dos tipos de imagenes, las tipo raster pierden calidad frente a las vectoriales.

Que es SVG

SVG es un lenguaje basado en XML que es un lenguaje de marcado como HTML. Todos ellos son de marcado pero con elementos diferentes.

Con SVG marcamos gráficos, no contenido. Por ejemplo con el elemento circle se dibuja un circulo.

Tambien incluyen funciones: feColorMatrix (transforma colores usando una matriz/paleta colores), animate (anima partes del grafico), mask (aplica una mascara sobre la parte superior de la imagen).

Dentro de la etiqueta de apertura y cierre del SVG se ponen elementos y atributos con valores para representar la imagen.

Crear un SVG escribiendo el código se puede complicar demasiado. Profesionalmente se usan programas/editores de graficos vectoriales como Inkscape (libre/gratuito, codigo abierto) o Illustrator (Adobe).

Un archivo SVG es más avanzado tecnologicamente que un JPEG por lo tanto tiene ventajas y desventajas:

Ventajas:

  • Tiene código, con ello ventaja en materia de SEO.
  • Se puede agregar CSS (estilo) y JavaScript (interactividad/funciones).
  • Desventajas

Desventajas:

  • Un SVG complejo demanda mucha potencia grafica
  • Puede ser más dificil de crear que un JPEG
  • No suele ser compatible con navegadores antiguos. Revisar en la web Can I Use?.

Por su parte las imagenes rasterizadas son mejores para imagenes complejas, como fotos.

En este sitio puede aprender a preparar SVG para la web.

Agregar SVG en el HTML

Se puede hacer de diferentes maneras

Una forma rapida: elemento img

Necesitamos un archivo .svg que va a ir en el atributo src, con alt (texto alternativo) y podemos agregar el ancho y alto.

Este metodo tiene pros y contras:

pros:

  • Es un metodo ya conocido, tambien atributos y valores.
  • Puede convertir el elemento img que contiente el SVG en un enlace con a. La imagen será un link.
  • El navegador guarda el cache, serán más rapidas las cargas.

contras:

  • No puede usar JavaScript en el SVG porque está en img y no en su propio elemento.
  • para aplicar CSS debe incluir estilo en el codigo del SVG.
  • No puede usar pseudoclases.

Solución a los problemas de compatibilidad en los navegadores antiguos.

Para navegadores que no admiten SVG, por ejemplo, el navegador de android 2.3 o IE 8 y anteriores.

En el elemento imagen ponemo la ruta del archivo SVG en src para navegadores antiguos y en el atributo srcset, ponemos el mismo svg para navegadores modernos.

Si bien es el mismo elemento con la misma imagen, son atributos distintos, el navegador aceptará el primero que entienda.

Puede usar SVG como imagen de fondo para un elemento del HTML, aunque no se podra usar JavaScript dado que usaremos la propidedad background de CSS.

Para navegadores antiguos usamos un imagen convensional para el fondo background: url ("fallback.png"). Para navegadores modernos usaremos un archivo svg y la propiedad background-image: url ("image.svg").

Si la imagen SVG no carga, puede ser que sea un problema con servidor y las imagenes SVG.

Agregando codigo SVG en el HTML: elemento svg

Podemos crear un SVG con codigo y este pegarlos o escribirlo directamente en el documento HTML.

Ventajas:

  • Ahorra una solicitud HTTP que puede reducir los tiempos de carga.
  • Podemos usar clases o id's para aplicarles estilo con CSS
  • Podemos usar interacciones como :focus y animaciones de CSS.
  • Podemos convertir SVG en enlace, envolviendo la imagen en el elemento a.

Desventajas:

  • Si usamos el SVG en distintos lugares, requerirá muchos recursos y mantenimiento.
  • El codigo SVG aumentará el tamaño del documento HTML.
  • No puede almacenar cache ya que la imagen es representada a traves de un código y no un archivo grafico. No reduciría timpos de carga.
  • Puede incluir un respaldo (fallback: propiedades de respaldo para compatibilidad en navegadores) en un elemento foreingObject aunque se descargaran las imagennes de respaldo.

Incrustar un SVG con iframe

Como podiamos insertar otras páginas con iframe, podemos insertar un documento SVG.

Usamos dos atributos src uno con un archivo SVG para navegadores modernos y otro con un .PNG para navegadores antiguos.

No es el mejor metodo para poner un SVG en una web.

desventajas:

  • La propiedad/archivo de respaldo solo funciona si el navegador carece de soporte para iframe.
  • No puede usar JavaScript a menos que el SVG y página activa/actual tengan el mismo Origin.
