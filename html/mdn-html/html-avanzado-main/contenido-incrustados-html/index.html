Contenido incrustado

Podemos ampliar la variedad de contenido en el sitio con los elementos iframe, embed y object. Con iframe podemos insertar otras paginas web con iframe (mapas, videos, etc), con embed podemos poner una aplicación externa (plug-in o complemento) que junto con object tienen una función parecida, se usan para incrustrar PDF, SVG, etc.

Elemento iframe

Nos permite insertar sitios web de tercermos como mapas, audio/video, sistema de comentarios, etc. Generalmente las herramientas de estas páginas se encuentran en el botón compartir, insertar, copiar código iframe.

Atributos para configurar un iframe:

  • allowfullscreen: Activa la pantalla completa para el elemento incrustado
  • src: Se escribe la URL de elemento que se quiere insertar
  • width y height: Indicamos el ancho y alto para el iframe.
  • Contenido de respaldo: Como hicimos antes agregamos un parrafo de aviso para los navegadores que no soporten iframe, junto con un enlace hacia la página dónde se encuentre el recurso.
  • : Activa una configuración para darle mayor seguridad a tu web, así poder usar el recurso externo.

*Se suele establecer el atributo src de los iframes con JavaScript para retrasar la carga de los iframe, para que la página carge más rapido.

Seguridad para el elemento iframe

Los atacantes pueden incrustrar iframe invisible en tu página o insertar tu página con un iframe en su página maliciosa. Esta tecnica de hackeo se llama Clickjacking.

Primero que nada, aseguremonos de que si incrustamos un recurso de una web; tenga el protocolo HTTPSHTTPS. Lo podemos ver al principio de la URL. El sitio que desarrolles tambien debería tenerlo. Github pages asegura tu contenido HTTPS, pero si estas usando un proveedor de hosting deberias consultarselo a esa empresa.

Segundo, el contenido de terceros generalmente tiene derechos de autor o una licencia que debe cumplir y dar credito de ello, como podemos ver en esta guía sobre buenas practicas de atribución. Cuando busques contenido de terceros asegurate de leer su licencia y pedir permiso para asegurarte de usar correctamente el contenido otras personas.

Tercero, usar siempre el atributo sandbox sin ningún valor. Por si solo, contiene todas las restricciones para cubrir las debilidades que tiene el atributo iframe y obtiene todos los permisos necesarios para hacer su trabajo.

Configurar CSP directives

CSP significa content security policy y provee HTTP Header que son un conjuto de metadatos que van en el head del documento, diseñados para mejorar la seguridad del documento cuando se recibe una llamada al servidor

.

Si configuramos en el servidor un X-Frame-Options header apropiado evitamos que otros sitios malicioso o no, usen el contenido con fines inapropiados.

Puedes leer la publicación de Frederick Braun, que tiene una visión más profunda sobre como aplicar este metodo.

Elementos embed y object

Son elementos que pueden incrustar complementos (tecnologia de terceros, no nativa a la web) como Flash, Applets de Java, videos, PDF. Estas tecnologias quedaron en deshuso, tal vez haya en proyectos de uso empresarial o intranet (herramientas de comunicación entre equipos en las empresas). Para insertar videos usamos su etiqueta o iframe, para insertar pdf es mejor usar un enlace.
