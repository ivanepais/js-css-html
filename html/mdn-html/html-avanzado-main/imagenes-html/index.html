Imagenes en HTML

Una web con solo texto es aburrida, por suerte se pueden poner imagenes y otro tipo de contenido multimedia. Son usadas para finas didacticos como dar al usuario una pista sobre el contenido, esquemas/resumen de información; puede ser más facil describir algo con una imagen que con palabras. Tambien son usadas para fines esteticos: las podemos encontrar en los encabezados de la web, de fondo en un titulo de algún articulo para la apariencia de la web,

Poner una imagen en una página web

Se utiliza el elemento img junto con el atributo src (source) en el cual debemos poner la ruta donde se encuentra la imagen.

Debemos recordar como se navegaba por la jerarquia de archivos para colocar las rutas adecuadamente.

Se recomienda tener una carpeta solo con las imagenes por temas de SEO.

No se recomienda poner imagenes de otros sitios por cuestión de copyright (a menos que tenga una licencia que permita el uso) y funcionalidad (las pueden borrar, cambiar, mover) que no sean las de nuestro proyecto, o sea, que esten en el mismo servidor que nuestro HTML.

Texto alternativo

El atributo alt es otro etiqueta de apertura del elemento img. Debe ser una descripción de la imagen, en caso de que no se pueda mostrar o cuando la página tarde en cargar el sitio completo.

Se incluye este atributo por cuestiones de accesibilidad, los lectores de pantalla de usuarios invidentes leen este atributo. Tambien se usa por cuestiones de funcionalidad, hay usuarios que desactivan las imagenes para reducir el uso de datos, otros usuarios aun usan navegadores de solo texto, solo verán la descripción. Los motores de busqueda pueden hacer coincidir el texto alternativo con las consultas de busqueda.

¿Qué se deberia escribir en el atributo alt? Depende de la imagen o preguntarse qué se pierde si la imagen no aparece.

  • Decoración: Para imagenes decorativas se debería usar CSS, con HTML solo deberiamos utilizar imagenes informativas. Pero si las hay, no hay que poner el atributo alt.
  • Contenido: En la imagen informativa se describe un texto alternativo breve que resuma su contenido.
  • Enlace: Al poner una imagen dentro de un enlace se debe seguir los tips para dar un texto claro a la imagen y describir la imagen con alt.
  • Texto: No deberias usar imagenes con textos/parrafos

En el apartado de accesibilidad de HTML en MDN, hay una guía de texto alternativo.

Anchura y altura

Los atributos width y height se usan para determinar el ancho y alto de una imagen desde el HTML. Pero esta tecnica no se debería aplicar, una buena practica es editar la imagen con un programa (como gimp u otro más simple) para conseguir un tamaño ideal para evitar usar muchas datos. Pero esto se ampliará más adelante.

Usar estos atributos no reducen el tamaño del archivo, solo esconde su tamaño.

Título de imagenes

Con el atributo title podemos agregar un titulo a la imagen, éste saldrá cuando el usuario pase el cursor sobre ella. Este funcionamiento ocasiona problemas de accesibilidad por eso no se tiene en cuenta.

La información que puede estar en title se puede poner como texto en un parrafo. Aunque hay etiquetas que cumplen esta función, como lo son: figure y figcaption

Agregar pie de imagen

figure hace de contenedor para la imagen y para figcaption que es el pie de imagen, en el que vamos a escribir el contexto de la imagen o otra descripción que necesite.

Figcaption y alt no deberian decir lo mismo, porque si la imagen no se muestra (por tardar en cargar), aparecerá el mismo texto.

Por otro lado, el elemento figure puede actuar de contenedor independiente para algo que:

  • Expresa un significado en forma compacta y facil de entender.
  • Se pueda poner en cualquier lado de la página.
  • Proporciona información esencial que da apoyo a la página.

Se aplica como contenedor de varias imagenes, un fragmente de código, audio, video, una tabla o a cualquier otra cosa.
