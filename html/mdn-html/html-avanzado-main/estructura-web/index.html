Estructura de una página web

La estructura de una página web depende de unos elementos de bloque que van a representar a un encabezado, un menú de navegación, el contenido principal, tambien un contenido secundario, un pie de página y más elementos que se usan para diseñar una web.

Partes basicas de una web

Son las partes que toda web debe tener a menos que este mal estructurada, sea para mostrar video/contenido multimedia, un juego o proyecto artistico.

Encabezado
Visualmente es una franja superior en horizontal que ocupa todo el ancho de la página. Puede tener un titulo, logotipo o un lema. Es una parte que no suele cambiar
Barra de navegación
Son los enlaces a las secciones principales del sitio formada por un menú con botones, enlaces o pestañas. Por cuestiones de diseño algunos prefieren que esten junto al encabezado, pero otros por cuestiones de accesibilidad prefieren que esten separados para que los lectores de pantalla puedan reconocer ambos elementos. Tampoco suele variar.
Contenido principal
Visualmente toma gran parte de la pantalla porque posee la mayor cantidad de contenido. Suele cambiar de página en página, a veces hay un articulo, otras veces hay un video. Puede tener todo tipo de contenido como noticias, productos, un juego; depende del sitio.
Barra lateral
Suele incluir enlaces, articulos relacionados con el principal, publicidad, un menú de navegación secundario, etc.
Pie de página
Es la parte inferior de la página, tiene una fuente pequeña para detallar cuestiones legales, copyright o información de contacto. Tambien es usada para contener enlaces de acceso rapido al contenido destacado por cuestiones de SEO que generalmente usan las empresas grandes.

Elementos de HTML para estructurar el contenido

Son los elementos adecuados que se usan para representar las partes basicas de una web. Los elementos semanticos describen su contenido para los navegador, motores de busqueda o usuarios no videntes que usan lectores de pantalla. Se describirán son los indispensables, más adelanate veremos más, pero en la referencia de HTML de MDN figuran todos detalladamente.

Encabezado: Representado por el header, son las ayudas introductorias o de navegación que se introducen en la parte superior de la web mediante iconos, enlaces, botones. No debe confundirse con los encabezados de titulo (h1-h6), ni estos con la etiqueta title que figura en el head del documento.

Menú de navegación: Representado por nav, dentro suele haber una lista no ordenada con los enlaces a las diferentes secciones de la web. Pueden ubicarse en el encabezado o debajo de él.

Contenido principal: Representado por el elemento main, agrupa el contenido principal de una página, debe haber uno por página y no debe estar anidado en ningún otro elemento. Generalmente contiene a otros elementos como:

  • article se usa para articulos de revista/periodisticos, posteos en bloq o plataforma, tarjetas de producto y otros elementos parecidos. La idea es que se use para elementos que puedan entenderse individualmente a pesar de pertenecer a un grupo, una tarjeta con la descripción de un producto o un articulo de un blog se entienden por si solos.
  • section agrupa contenido que se desarrolla a partir de un titulo o tema con un encabezado, despues pueden seguir articulos que desarrollan temas relacionados. Podemos dividir articulos en distintas secciones o viceversa. Tambien es usado para agrupar comandos/funciones/botones que no tienen otro elemento para representarlos como puede ser una lista de resultados que da un buscador.
  • div es elemento no semantico usado para propositos que no disponen de una etiqueta especifica y para aplicar CSS. Si se quiere mantener un sitio orientado a la semantica, usarlo con cuidado.

Barra lateral: Representado por el elemento aside presenta información adicional con enlaces a modo de referencia del contenido principal o contenido parecido.

Pie de página: Representado por el elemento footer, generalmente contiene enlaces a las secciones más importantes o información relacionada con el autor del sitio, información legal, etc.

Elementos no semánticos

Cuando no existe elementos para agrupar cierto tipo de contenido se puede usar los elementos no semánticos.

Existen elementos no semanticos en bloque y en línea como div y span, generalmente se usan con el atributo class para llamarlos desde el CSS dandole estilo o desde JS aportandole funcionalidad.

Planear un sitio web

Se pensar en el contenido de un sitio web, todas las páginas que se necesita y la forma de disponer las conexiones entre ellas para producir la mejor experiencia de usuario a los visitantes. Se lo conoce como Arquitectura de la información, se lo utiliza en proyectos grandes.

Para un sitio pequeño podemos planificar una web apuntando lo que necesitamos en un papel.

Por ejemplo:

  1. Habrá varios elementos en común en muchas o en todas las páginas del sitio (según los elementos), como un menú de navegación, pie de página, etc. Piense en que necesita el sitio y anota lo que debería tener en común todas las páginas.
    • Caracteristicas comunes del sitio: titulo, logotipo, contacto, derechos de autor, terminos y condiciones, politicas de privacidad, seleccion de idioma y politicas de accesibilidad.
  2. Dibuja el esquema o el diseño de la página como las partes principales
    • Un encabezado, un area de contenido principal, barras laterales (opcional) y un pie de página.
  3. Ahora, haz una lista completa con todo el resto de contenidol que no sea común en todas las páginas, relacionada con la tematica del sitio.
  4. Ordenar el contenido anterior en grupos para conectar que partes podrían ir juntas en las diferentes páginas.
    • Haz categorias y pone los elementos que deberián en cada una.
  5. Esquematiza un mapa de tu sitio, dibuja un globo para cada página del sitio y dibuja lineas que identifiquen el flujo de datos entre las distintas páginas. La página de inicio estará en el y enlazará con el resto de las páginas. Tambien puedes hacer notas para el diseño de los elementos.
    • El mapa tiene que mostrar la página de inicio con todo su contenido, así con las demás páginas.
