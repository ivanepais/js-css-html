Debuggear HTML

Cuando falla algo en el HTML y no podemos descubrir a simple vista a dónde está el error, podemos usar las herramientas de desarrollador que nos provee el navegador.

Debuggear no debe asustarnos

Cometer errores de programación es muy común en la programación, de hecho a veces, los programadores se la pasan leyendo y arreglando código que escribiendolo.

HTML no es tán dificil como otros lenguajes y corregirlos es sencillo. Además HTML es permisivo.

Código permisivo

Cuando escribimos código solemos cometer dos tipos de error:

  • Errores sintácticos: Son errores de escritura o sintaxis (elementos, comillas, llaves), suelen ser faciles de arreglar si entendemos el mensaje de error que nos figura en el inspector de código.
  • Errores lógicos: Aquí la sintaxis es correcta, pero el código no hace lo que debería y hace que el programa/sitio no funcione adecuadamente. Son los más dificiles de solucionar, no hay mensaje que nos pueda ayudar o guiar. La solución depende de nuestra intuición e ir buscando el error del código, usando las herramientas del inspector de código.

HTML no suele producir errores sintácticos, el navegador los corrige internamente. Esto se debe a que en los comienzos de la web, era más importante que la gente pueda publicar contenido que los errores de escritura que pudiera tener el código.

Errores en HTML e Inspector

El navegador puede corregir los errores del código fuente HTML.

Validar HTML

Cuando los documentos son complejos y grandes, no es tan facil darse cuenta dónde están los errores.

Para esto podemos usar la herramienta de Validación de HTML de la organización w3 que se encarga de definir como se debería usar HTML, CSS y otras tecnologías.

Para validar, podemos usar la dirección del sitio, un archivo html o poner el codigo html completo, en el campo correspondiente.

Interpretación de los mensajes de error

A veces las recomendaciones no son las adecuadas para nosotros. Los errores pueden producir un efecto domino, si arreglamos unos cuantos y recargamos la página, se arreglaran otros automaticamente.
