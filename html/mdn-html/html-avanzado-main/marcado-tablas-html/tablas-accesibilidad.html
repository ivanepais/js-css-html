Tablas y Accesibilidad

Tratamos algunas funciones más avanzadas sobre las tablas, como subtitulos/resumen, agrupación de filas en secciones de encabezado, el cuerpo y pie de la página. Tambien conceptos sobre la accesibilidad en las tablas.

Subtitulos en la tabla: elementos caption

Con caption ponemos un titulo a la tabla, justo debajo de la etiqueta de apertura del elemento que la define; table.

El contenido de caption debe ser una breve descripción de la tabla. Es más util para accesibilidad, para un usuario ciego, su lector de pantalla leera primero este elemento antes del contenido. Así puede avanzar rapido sobre la web y pasar lo que le interesa.

Estructura de una tabla

Son utiles para aplicar CSS y diseñar estos elementos, podemos hacer que en una tabla extensa se repitan el encabezado o el pie de página al desplazarse por el contenido. Por si mismo no ayudan en accesibilidad.

El elemento thead debe contener a los encabezados de la tabla. Si hay colgroup y col, debe ir debajo de estos.

El elemento tfoot debe contener a la fila final que generalmente son datos resaltados que resumen todo el contenido anterior. Una cosa curiosa es que el navegador va a mostrar esta fila, al final de la tabla, aunque en el html este puesta debajo de los encabezados por ejemplo.

El elemento tbody debe contener a todos los elementos que no pertenecen a la fila de cabecera de la página ni a la fila del pie de página.

Desde el lado de CSS llamanos a estos elementos para aplicar fondos, colores, fuentes, etc.

Anidar tablas

Se usa si es muy necesario importar contenido de otra tabla. No se recomienda anidar tablas, pero si se hace, se pone la estructura completa de las dos tablas.

Elementos para darle accesibilidad a una tabla

Una tabla bien hecha, visualmente tiene resaltados los encabezados para relacionar y entender rapidamente los datos. Esto se puede hacer facilmente con CSS, pero para darle accesibilidad necesitamos hacer con elementos o atributos.

Usar encabezados de columna y fila

Se aplican con el elemento th, los lectores de pantalla leeran estos encabezados y los usuarios no videntes podrán hacer las misma asociaciones que cualquiera que vea la tabla.

Atributo scope

scope se añade a los encabezados (th) para indicar si es un encabezado de fila o de columna, con sus valores: row y col.

Tiene otros valores como: rowgroup y colgroup para encabezados que contiene a otros.

Alternativa a scope: atributos id y headers

Se usan los atributos id y headers para crear asociaciones entre los encabezados y sus celdas.

  1. Añadimos un id unico que represente el contenido del encabezado th
  2. Añadimos el atributo headers a cada elemento de celda (td), headers debe contener una lista de los id de los encabezados de esa celda dejando espacio.

Tiene muchisimo codigo y no admite errores porque confundiría a los usuarios no videntes. El metodo con scope basta para la mayoria de tablas.
