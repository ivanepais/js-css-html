Conceptos basicos de HTML

HTML ("hypertext markup lenguage") no es un lenguaje de programación, con él no se puede escribir lógica, resolver problemas complejos. Es es un lenguaje de marcado para estructurar el contenido de la web así los navegadores entienden cada elemento. Se utilizan elementos de marcado se utilizan para marcar, delimitar las partes del contenido del sitio. Los elementos de marcado tienen etiquetas para encerar el contenido de la página.

Anatomia de un elemento HTML

Son las partes que posee cualquier elemento de HTML.

  • Etiqueta de apertura: Marca donde comienza el elemento por eso se encuentra antes del nombre de este. Se trata de un parentesis angular.
  • Contenido del elemento: Puede ser texto o una imagen, un link, etc.
  • Etiqueta de cierre: Se distingue de la etiqueta de apertura por la barra vertical antes del nombre del elemento. Verá que algunos elementos no tienen esta etiqueta.

Elementos anidados

Son elementos dentro de otro, debemos tener en cuenta que la etiqueta del primer elemento debe estar al final, conteniendo a todos los elementos.

Elementos en bloque y elementos en línea

Son las dos categorias de elementos más importante de HTML, porque afecta a la visualización de la página. Debemos conocer sus fundamentos para evitarnos problemas con el CSS.

  • Los elementos en bloque aparecen uno debajo de otro, en una línea nueva. Son elementos estructurales de la página como titulos/encabezados, parrafos, listas (ordenadas, desordenadas y descriptivas) menús de navegación o un footer (pié de pagina). Un elemento en bloque puede estar dentro de otro, pero no de un elemento en línea
  • Los elementos en línea son los que estan contenido en un elemento para remarcar, señalar algo del contenido de una web en una misma línea. Como son los links, o elementos de enfasis como cursivas o negrita.

En HTML5 se redefinió y amplió las categorias de elementos.

Elementos vacios

Son los elementos que no tienen etiqueta de cierre como puede ser el elemento que se usa para poner las imagenes en la web: img.

Atributos

Los atributos dan funciones e información adicional a un elemento de HTML. No afecta al contenido sino que ayuda al visitante de la página.

Para que funcione debe cumplir con estás condiciones:

  • Se colocan dentro de la etiqueta de apertura.
  • Deben estar separados con un espacio desde el nombre del elemento, si hay más atributos, tambien.
  • El nombre del atributo va seguido de un igual (sin espacios).
  • Tienen diferentes valores que van dentro de dos comillas dobles que se ponen despues signo del igual.

Por ejemplo, un elemento de link o enlace puede tener los siguientes atributos para darle más información al usuario.

  • href: Su valor será la dirección del sitio web al que apunta.
  • title: Su valor será la descrición del sitio, podría ser algo como "La página de inicio de Google". Aparecerá una solapa con está información si pasamos el cursor encima del link.
  • target: Tiene como valor a _blank que abrirá abrira el enlace en una pestaña nueva. Si omitimos este atributo, se abre el otro sitio en nuestra ventana actual.

Atributos booleanos

Son atributos que no necesitan valor, solo con su nombre funcionan.

Por ejemplo, el elemento imput es un campo para entrada de datos, decimos que este campo aceptará texto (con el atributo type="text"). Si al lado escribimos el atributo disabled desactiva el elemento completo y el usuario no puede ingresar datos.

Otros atributos booleanos son: checked (marca al elemento con un signo de verificación), defer ( retraza la carga del código seleccionado), selected (selecciona un elemento despues de cargar la página). Ahora no debe preocuparnos ya que son más avanzados.

Tipos de comilla en los atributos

Se pueden usar las comillas dobles o simples, lo importante es no olvidarse de colocarlas o mezclarlas porque inutilizan el código.

Anatomía de un documento HTML

Debemos combinar los elementos individuales para formar una página de HTML completa.

  1. !DOCTYPE html: Es un elemento que está desde los inicios de la web, hoy en día es un legado pero se incluye para que todo pueda funcionar correctamente.
  2. HTML: Es el elemento raíz de una página web, contiene a los demás elementos que usamos.
  3. head: Van los elementos que no son visibles en la web, como metadatos, las hojas de estilo del documento, etx.
  4. meta charset="utf-8": Establece que el documento utilizará la códificación de caracteres UTF-8 para que el documento acepte los caracteres de la mayoría de idiomas conocidos.
  5. body: Hace visible los elementos y todo el contenido que queremos mostrar y que pueden ver los visitantes del sitio.

Espacios en blanco en el HTML

En cualquier elemento de HTML se contará solo un espacio, por más que agregemos miles; solo se mostrará uno.

La legibilidad de un texto y como están formateado y separados los elementos desde el HTML son igual de importantes, hace al documento mantenible para los desarrolladores.

Por eso no tiene sentido agregar tantos espacios desde el HTML. Los espacios entre los elementos que vemos en cualquier página están dados por el CSS. Por ahora no deberiamos preocuparnos por este.

Caracteres especiales en HTML

Son los caracteres que al querer escribirlos desde el documento HTML chocan con el código del HTML porque son parte de su sintaxis, por este motivo no se logran mostrar en la página.

Los siguientes signos se componen de un caracter inicial que es & (signo de ampersand), una inicial y un caracter final que es ; (punto y coma). Así logramos mostrar estos signos en una web. Un articulo de Wikipedia detalla más acerca de estos.

Por ejemplo, si queremos mostrar en la web la etiqueta parrafo, debemos escribir desde el HTML: <p>.
