Fundamentos del texto en HTML

El texto por si solo no tiene significado para el navegador, ni para los buscadores; además de que no podríamos aplicarle CSS ni JS. Para esto necesitamos a los elementos que fueron creados para darle semantica a un texto. Los elementos de texto y en general le dan estructura al contenido, cualquier articulo necesita de un titulo, parrafos, enlaces, etc. Por otra parte esto tambien ayuda a las personas que necesitan de un lector de pantalla, es la única manera que se de cuenta cúal es el titulo, los parrafos, las palabras resaltadas, frases, citas, los enlaces, etc.

Encabezados y párrafos

Cuando necesitemos un titulo o un parrafo usamos los elementos p y h1, h2, h3, que son los demás elementos de encabezado.

Es una buena practica usar solo un h1 por página que sería titulo principal, h2 será el subtitulo y h3 el subtitulo del subtitulo. Otra manera de verlo es que los encabezados representan distinto nivel de contenido que usaremos para ordenar un articulo. Tambien ver los encabezados y los párrafos en las tiendas online en las que hay cartas/recuadros que representan la descripción de un producto.

Otro cosa a tener en cuenta es que no debería usarse CSS para hacer que un párrafo o algún texto se parezca a un título. Directamente usamos el elemento que fue creado para tal fin, lo mismo aplica para los demás.

Crear una estructura jerarquica

Depende lo que necesita el texto o lo que se necesite redactar. Debe tener lógica y sentido comun al seguir el orden de los encabezados, no se puede tener un nivel de encabezado menor que agrupe contenido con un encabezado más grande.

Algunas consideraciones:

  • Se recomienda tener un solo h1 por página, en este nivel se ubicarán los demas encabezados.
  • No se debería usar un titulo con h3 para agrupar contenido con subtitulos con h2
  • De h1 hasta h6 se recomienda no usar más de 3 niveles de encabezados a menos que sea necesario uno más. En caso de necesitar muchos niveles, es mejor separar el contenido en varias páginas, puede resultar incomoda la navegación por el contenido para el usuario.

Ventajas de estructurar el documento

Hace que la página se más legible para el usuario, este siempre buscará la información rapidamente y si no la encuentra se irá.

  • Los usuarios tienden a escanear la página para encontrar contenido relevante, ademas suelen pasar muy poco tiempo en una página web. Si no ven nada útil se van a otro sitio.
  • Los motores de busqueda toman los encabezados como palabras clave para posicionar la web SEO en sus resultados. Sin encabezados, la página tendrá un rendimiento bajo en terminos de SEO según la guía de Google.
  • Para aplicar CSS y JS se necesita de los elementos de HTML.

Listas

Usamos las listas para describir un conjunto de datos o info relacionada a un tema en particular, seguir instrucciones, tareas u obligaciones que tenemos, etc. Hay tres tipos de listas: ordenadas, no ordenadas y descriptivas. Todas las listas deben llevar un elemento de lista li en donde va la información.

Lista ordenada

Se crea con el elemento ol, al principio de la lista veremos que está ordenada por numeros.

Lista no ordenada

Se crea con el elemento ul, al principio de la lista veremos unos puntos antes de la información.

Lista descriptiva

Puede ser usada para describir conceptos más complejos y extensos como puede ser abordar un tema completo, con encabezados y definiciones. Se compone de tres elementos: dl crea la lista, dt da el titulo al concepto/definicion y dd descripción/contenido.

Listas anidadas

Podemos poner una lista dentro de otra creando una lista ordenada, no ordenada o descriptiva dentro de un elementos de lista (li).

Enfasis e importancia en el texto

Destacamos un palabra o frase que nos parece importante o diferente.

Enfasis

Con el elemento em podemos poner enfasis al texto para modificar/resaltar levemente el significado o importancia de alguna cuestión o hecho que aparezca en el texto, el texto se inclinado. Además los lectores de pantalla cambiarán el tono al leer el enfasis. Son elementos semanticos, debemos tenerlos siempre en cuenta para la accesibilidad (inclusividad) y el SEO (posicionamiento en buscadores).

Importancia fuerte

A las palabras muy importantes le damos el enfasis con negrita, en HTML esta función depende del elemento strong. El tono de los lectores de pantalla se incrementará, si no queremos esto pero si queremos que el texto tenga negrita; podemos usar otras propiedades de CSS mediante un elemento span a la palabra o usar el elemento b (si corresponde) explicado más adelante.

La experiencia de lectura se enriquese resaltando las palabras más o menos importantes, logramos que el usuario encuentre lo que necesita rapidamente o le ayude a quedarse con los conceptos importantes.

Cursiva, negrita y subrrayado

HTML nos proporciona para estas funciónes cuando no queremos resaltar ninguna importancia y solo queremos lograr este aspecto en el texto pero no deberían usarse porque no son semanticos, no se tienen en cuenta en la accesibilidad y SEO.

Aunque la nueva especificación de HTML5 refinio su uso solo en estas ocaciones:

  • i (italica o cursiva): Se usa en un texto/oración que contenga extrajerismos (palabras en otro idioma), calificaciones taxomomicas (categorizar, ordenar, estructurar organismos vivos), conceptos tecnicos (tecnicismos/nombres cientificos), un pensamiento.
  • b (bold/negrita): Se usa para palabras claves, nombres de productos, frases principales.
  • u (subrrayado): Se usa para nombres propios, errores ortográficos.

Entonces, ¿Qué elementos debemos usar para resaltar partes del texto? Podemos tener en cuenta que una palabra puede ser importante una vez coomo concepto general o global y la resaltamos con em o strong pero despues si se repite podemos usar i o b o span con css para llamar la atención del usuario y no se pierda en el texto. No deberíamos llenar el texto de em o strong. Si tenemos otras dudas, es mejor atenerse a la especificación.

Las etiquetas em y strong no se usan por las cursivas o la negrita (es solo su CSS por defecto), sino, por el nivel de importancia en el texto. Si queremos un efecto parecido y ademas al texto al que queremos aplicar no cumple con las reglas para i, b o u; es mejor usar un span con CSS.

Conclusión

Cada apartado del texto tiene un elemento de HTML que debemos usar, como lo es en general para cualquier apartado de un sitio web. Para esto debemos hacernos preguntas sobre qué elemento tiene más sentido usar para la cosa que queremos hacer.
