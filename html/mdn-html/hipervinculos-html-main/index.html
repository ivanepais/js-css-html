Los hipervínculos

Están desde el comienzo de la web, hacen que la web sea la web porque con tan solo pulsarlos podemos navegar por cualquier sitio; podríamos decir que son la web, sin ellos no podríamos visitar ningún sitio.

Anatomia de un enlace

Al proveerle una dirección (URL) podemos vincular en sitio con otros recursos internos y externos.

Un enlace se crea con el elemento s y en la etiqueta de apertura tiene el atributo href, adentro va la dirección.

Atributo title en un enlace

Debemos describir resumidamente el contenido de la página enlazada. Mostrará un recuadro pequeño con la descripción dada al poner el cursor sobre el enlace.

Para los dispositivos moviles no funciona, si esta descripción es muy importante para el uso, debemos escribirla en el enlace.

Convertir una imagen en un enlace

Cualquier cosa se puede convertir en un enlace si se lo incluye dentro de sus etiquetas.

Encerramos el elemento img, dentro de a. La etiqueta de apertura de imagen tiene el atributo src en el tambien indicamos la dirección donde está la imagen

Buenas practicas para el uso de enlaces

Hay una serie de comportamientos de los usuarios y buscadores con respecto a los enlace a tener en cuenta:

  • Los usuarios de lectores de pantalla suelen saltar de enlace a enlace y los leen si contexto.
  • Los buscadores utilizan el texto de los enlaces para posicionar la página, es recomendable utilizar palabras claves al escribir un enlace.
  • Los visitantes hacen un vistazo rapido para leer lo que les interesa, mirán lo que se destaca de la web como pueden ser el texto de los enlaces.

Tips para escribir enlaces:

  • NO poner la URL como parte del texto del enlace porque son larguisimas y no tienen sentido. Se describe muy resumidamente hacia donde va el enlace.
  • No escribir "link" o "link a", tampoco "enlace" o "enlace a" porque es redundante y no aporta nada al usuario, ni es una palabra clave para los buscadores.
  • Los enlaces generalmente están dentro de un contexto desarrollado en los párrafos de la página en el que es dificil que se repita el texto que hay en ellos. Pero en el caso de que los enlaces no tengan un contexto, evitar que su texto este repetido. Ejemplo, al costado izquierdo de la página suele haber link, debemos evitar que esten escritos como "haz click aquí" repetidas veces.

Indicar claramente los recursos no HTML

Se refiere a los enlaces que no llevan a una página normal sino que llevan a una página que abre/descarga un un recurso pesado como puede ser un pdf, juego, una descarga. Indicar si es muy pesado (o se peso) en el texto del enlace.

Ir hacia una parte de la página

Con el atributo id podemos movernos por los encabezados o secciones de una página.

El primer paso es describir el nombre del encabezado en el valor del atributo (sin espacios/con guiones), el segundo paso es crear un link con la dirección de la página + #valor-id/seccion-de-la-página.

En el texto de este link vemos una referencia sutil a la sección a la que queremos enviar al usuario.

Tratamiento para URLs y rutas locales y externas

Las URLs (Uniform Resource Locator) definen donde esta situado un recurso en la web.

Las URLs utilizan rutas para encontrar los archivos, estas rutas son una cadena continua de directorios. El directorio que usamos para contener y crear otros archivos y carpetas se denomina directorio raiz.

Cada uno de estos archivos y carpetas forman una nueva ruta con su nombre separadas por una barra vertical. Los archivos index.html generalmente son las página de inicio de cada sección de la web y estas son las carpetas contenedoras con las otras páginas.

Estructura y navegación por las URLs

Es esencial conocer el uso de las rutas cuando usamos los enlaces, podemos navegar por el mismo directorio, subir o bajar de directorio usando distintos caracteres. En el atributo href del elemento a, irán las rutas de los archivos y carpetas.

Si leemos de izquierda a derecha

  • Por un mismo directorio: Para enlazar otros archivos sueltos que esten en el mismo directorio del archivo que estamos usando, solo tenemos que escribir el nombre de ese archivo.
  • Subiendo un directorio: Escribimos el nombre de la carpeta que se encuentra en nuestro directorio, seguido de una barra y por ultimo el nombre del archivo. Por ej: nombre-de-carpeta/archivo.html.
  • Bajando un directorio: Si nos situamos en el archivo de arriba de todo y en este queremos enlazar algún documento que esta suelto abajo, ../archivo.html. Si el archivo al que queremos llegar esta dentro de una carpeta, ../nombre-de-carpeta/archivo.html.

Si entendemos que página de inicio está arriba de todo: Con ../nombre-archivo-o-carpeta subimos de directorio y con /nombre-archivo-o-carpeta bajamos.

URLs relativas y absolutas

Se diferencian por su ubicación y tamaño (partes de una URL).

  • URL absoluta: El enlace incluye el protocolo (HTTP/HTTPS) y nombre de dominio (ej: google) hasta el archivo/página que desea llegar. Siempre apuntará a la misma dirección sin importar desde donde se utilize
  • URL relativa: El enlace comienza desde la posicion actual hasta el archivo donde queremos llegar, son como los enlaces que usamos para navegar entre directorios. Si cambiamo de lugar los archivos, no apuntará al mismo sitio.

Cuando usar enlaces relativos y absolutos

Para navegar entre directorios usamos las URLs relativas y para los enlaces que llevan a otra parte del sitio web, usamos URLs absolutas.

Crear un menú de navegación

En el directorio tenemos la página de inicio (index.html) y las otras secciones.

En cada página creamos una lista no ordenada, agregamos los list item necesarios según las secciones del sitio.

Ponemos los enlaces a cada una de estas, pero no ponemos el que va a la página que estamos usando.

Enlace a un correo electronico

En el atributo href de un enlace podemos poner una direccion de correo. Antes del correo va la palabra mailto:. Finalmente en la descripción del enlace ponemos hacia quién va ir el mail.

Cuando el usuario haga click en el enlace, el navegador abrirá el cliente de correo electronico o el gmail para que redacte el mail.

Si omitimos la direccion de correo y solo ponemos href="mailto:", se abrirá el cliente de correo y los usuarios podrán elegir el destinatario que quieran.

Incluir asunto, copia, copia oculta y cuerpo del mensaje

subject (asunto), cc (copia -otro mail-), bcc (copia oculta) y body (cuerpo para un breve mensaje). Para usar estos campos en la url, se necesitan unos caracteres especiales.

El Código porcentual se usan para los espacios, el signo de interrogacion (?) es para separar las URLs de los campos (subject, cc, bcc, etc), el signo de ampersand (&) es para separar los campos del enlace "mailto:".
