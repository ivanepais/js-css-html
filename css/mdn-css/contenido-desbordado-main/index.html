Capitulo 6: Contenido desbordado

Sucede cuando hay más contenido del que la caja puede soportar.

Desbordamiento

Podemos expandir o restringir el tamaño de una caja con width y height o inline-size y block-size. Es bastante frecuente en las compilaciones de CSS.

Perdida de datos

CSS trata de evitar que el contenido se pierda cuando no entre en una caja, por eso hace que se desborde.

El error es darle altura a la caja y no dejar que se adapte el contenido.

Propiedad overflow

Con esta, podemos controlar como se comporta el desborde de contenido. Su valor predeterminado es visible. Si queremos que se oculte usamos hidden hace que los datos se pierdan para el usuario, lo contrario a visible. El valor scroll agrega una barra de desplazamiento horizontal y vertical a la caja. La propiedad overflow-y: scroll agrega una barra de desplazamiento vertical. Los mismo con overflow-x: scroll agrega una barra horizontal, aunque no es recomendada para las palabras largas, es mejor investigar "word-break" y "overflow-wrap". Podemos utilizar dos valores en la propiedad overflow. El valor overflow: auto añade la barra de desplazamiento que se necesita cuando hay desborde.

El error está en que una fuente tan grande necesita más espacio o eliminamos el ancho fijo o usamos las propieades para controlar el desbordamiento.

Usar valores para controlar tiene efectos en el navegador y en la caja.

Algunos valores proporcionan barras de desplazamiento.

BFC (block formatting context)

El contexto de formato en bloque se convierte en un contenedor para tu caja, cuando se usa overflow. El contenido de adentro no sale y tampoco el contenido por fuera de esta caja puede superponerse.

Consejo

Cambiar el valor predeterminado de overflow tiene que reservarse para muy pocos casos. Se debe probar el diseño con grandes cantidades de contenido así como pequeñas cantidades, cambiar el tamaño de la letra para asegurarnos que el diseño soporte estos cambios.
