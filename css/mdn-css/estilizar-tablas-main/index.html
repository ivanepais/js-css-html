Estilizando tablas

Si no aplicamos estilos a las tablas son dificiles de leer y esteticamente no tienen identidad.

Conceptos basicos

Cada tabla se define con dos etiquetas table de inicio y fin. La etiqueta tr forma una fila, td forma las celdas de las tablas. Si no se usan encabezados no se puede distingir el contenido que clasifica los datos, entonces, utilizamos la etiqueta th para dar un encabezado a la celda.

Dentro de th y td va el contenido/texto para estas celdas.

  • Si ponemos una fila (tr) y dentro varios encabezados (th), tendremos una fila de encabezados horizontal o "columnas".
  • Si hacemos una fila (tr) y dentro ponemos un solo encabezado (th) y seguido ponemos varias celdas normales (td), tendremos un encabezado vertical con sus datos. Si lo hacemos varias veces tendremos una fila vertical de encabezados.

Expansión/unificacion de filas o columnas

Para celdas horizontales se usa el atributo las etiquetas llamado colspan y para las verticales se usa rowspan, ambas aceptan un valor numerico positivo y se cuenta la celda que se quiere expandir. Hay que tener cuidado con las celdas superpuestas si se usan estos atribujos seguidos, tambien si se usa "rowspan" el codigo es diferente a como se muestra.

Conceptos avanzados

Atributo summary

Se utiliza para dar una descripción precisa sobre el contendio de nuestra tabla. Es una funcion de accesibilidad, para los usuarios con lector de pantalla. Se ubica en la etiqueta de apertura table.

Titulo

Es la descripcion de la informacion que se presenta en la tabla, se usa la etiqueta caption al principio de la tabla.

Atributo span y scope

Un atributo span sirve para aplicar un estilo creado en col y colgroup (explicado más adelante) a las demas columanas, segun a cuantas se le indique. Por eso, acepta valores positivos. Aunque no es la única forma, se puede hacer lo mismo con la etiqueta col.

El atributo scope indica qué celdas afecta una celda de encabezado (solo para th), solo se usa en th y los valores pueden ser col, row, colgroup, rowgroup. . Los valores col y row indican que el encabezado provee info para el resto de las celdas en columna o fila que esté presente. Se usa para distingir con que celdas o columna de datos estan relacionadas con los encabezados. Ej: si un encabezado pertenece a una columnas se pone "col", si pertenece a una fila se pone "row".

Estructura para los datos

Ej: thead se usa para contener las filas horizontales de encabezados o "columnas", tbody se usa para encabezados verticales y sus celdas de datos horizontales. Cuando hay info que necesita distingirse con estilos, podemos crear dos tbody, sino no hace falta. La etiqueta tfoot se usan para las filas de resumen o fian.

Estilo a las filas o columnas

Colgroup

Es para una columna completa de datos, son etiquetas col y colgroup. La etiqueta "colgroup" sirve para aplicar estilos a un conjunto de columnas. Se escribe al principio de la tabla, despues del titulo. Las columnas se cuentan de izq a der y se usa junto al atributo span.

Ej: si tenemos 5 columnas (no importan las unificadas), y queremos hacer tres grupos de columnas, o sea; la primera columna queda sola porque no esta relacionada con el segundo y tercer grupo de columnas (formadas cada una por dos columnas) estas tiene datos distintos por eso las separamos. Aunque una columna este sola, debemos declararla como colgroup vacio para que las demas tengan sentido. Dentro de las etiquetas aplicamos estilo al grupo de columnas que queremos diferenciar.

Etiqueta Col

Representa a las columnas en las tablas de HTML, se escribe dentro de colgroup al inicio de la tabla o despues del titulo si es que necesita. Se usa para dar un conjunto de atributos y estilos a un grupo de celdas. Mediante el atributo span podemos especificar las columnas afectadas por los mismos atributos y estilos. Si creamos varios col es porque necesitamos varios estilos para las distintas celdas o grupos. De la misma forma que funciona colgroup. El fin de col no es semantico al relacionar datos con columnas, solo es estitico o para llamar la atención a los usuarios.

Resumen de una tabla

Comienza con la etiqueta table, dentro va el atributo caption que describe el contenido para accesibilidad. Luego viene el titulo (etiqueta title), si hace falta. Despues agrupamos las columnas con colgroup y col para definir caracteristicas y estilo a otras columnas, junto con el atributo span que determina cuantas columnas va a tener el mismo estilo. Seguimos con los encabezados th junto al atributo scope para definir que si los encabezados (exclusivamente) incluyen a fila o columnas con los valores row, col, rowgroup, o colgroup. Los valores "row y "col" es cuando un encabezado tiene columnas o filas normales. Los valores rowgroup y colgroup se usan cuando un encabezado mayor tiene encabezados menores y a su vez estos tienen datos en filas y columnas (scope row y col). Finalmente usamos las filas (tr) y columnas/celdas normales (td).

Aplicar estilos se puede hacer de varias formas a traves de la etiqueta style en el html: 1) Aplicando estilo a los encabezados (th), 2) Aplicando estilo con "colgroup", 3) Aplicando estilo con "col", aunque los ultimos tienen propisitos distintos como ya lo explicamos. Junto con los demas atributos explicados.

Consejos para el diseño de tablas

Un resumen del diseño que acabamos de crear:

  • Manten el marcado simple y las cosas flexibles con el uso de porcentajes para que el diseño sea responsivo.
  • Usa table-layout: fixed en la etiqueta table para permitir poner ancho en los encabezados
  • Define border-collapse en la tabla para eliminar los bordes de las celdas.
  • Usa las estructuras thead, tbody, tfoot para dividir la tabla y poder aplicar estilos más completos.
  • Usar distintos colores para las filas así facilitar la legibilidad.
  • Alinear los textos de las celdas y columnas para ordenar los elementos y facilitar la lectura de la tabla.
