Capitulo 3: Estilo a los enlaces

Hay que tener en cuenta algunas pseudoclases que se usan para los links, hay una lista que puedes revisar. Tambien puedes repasar como manejar los enlaces hacia archivos locales y hacia la web.

Los estados en un enlace

Pueden diseñarse usando las siguientes pseudoclases

  • Link (no visitado): Estado predeterminado de un enlace, se pone con la pseudoclase :link.
  • Visited: Cuando un enlace fue clickeado cambia de color, se puede hacer eso con :visited.
  • Hover: Cuando a un enlace se le pasa el cursor, se lo remarca para distinguirlo con :hover, puede cambiar colores, la forma del cursor, etc.
  • Focus: Si un elemento o enlace tiene foco, el usuario puede usar la tecla tab para llegar a él. Se hace con :focus.
  • Active: El enlace puede cambiar de apariencia o color cuando se hace click, se puede ver en un enlace que tenga esta pseudoclase :active si se mantiene apretado el click. A diferencia de :visited que se activa despues del click, :active funciona al momento.

Enlaces predeterminados

Por defecto vienen:

  • Subrayados
  • Los no visitados (:link) se ven azul.
  • Los visitados (:visited) se ven violetas.
  • Al pasar por arriba un enlace (:hover), el cursor cambia de forma.
  • Si tiene foco (:focus) se ve un contorno alrededor del link apretando tab y así se va cambiando (Windows).
  • Los activos (:active) se ponen de color rojo (se distingue si se mantiene apretado el click derecho).

Para el formateo de enlaces se recomienda:

  • Subrayar los enlaces, pero no otros elementos; sino, destacarlos de otra forma
  • Aplicar :hover para resaltarlos de alguna manera.

Los estilos por defecto se pueden cambiar usando:

  • color para el texto.
  • cursor para el tipo del cursor (no se recomienda deshabilitarlo).
  • outline es el contorno del texto, similar a un borde. Es ayuda de accesibilidad, no se recomienda desactivarlo.

Creando enlaces

En la hoja de CSS hay que seleccionar al enlace y escribir una pseudoclase (si se desea).

El orden de los pseudoclases importa, así un enlace va tomando la caracteristica de estas y puede funcionar correctamente.

El orden es el siguiente: L (:link). V (:visited). F (:focus). H (:hover). A (:active).

Iconos en los enlaces

Primero tenemos que usar un selector de atributo que es href con una palabra que tenga, en este caso: "html". Utilizamos la propiedad abreviada background, su valor url() para insertar la dirección de la imagen; usamos no repeat para que aparezca una sola copia y luego especificamos una posición adecuada para la imagen como 100% y 0 (derecha superior) o right top.

Para el tamaño de la imagen usamos background-size con 16px de ancho y alto.

Aspecto de botón en un enlace

La pseudoclase :hover se puede usar para señalar (si se desea) distintos elementos en la web como parrafos o listas, etc.

Otra cosa comun es dar a los enlaces forma de botón, para un menu de naveción que contiene una lista de enlaces con botones.

Propiedades interesantes que se pueden usar:

  • Utilizamos display para cambiar el comportamiento de los elementos a y li.
  • En la 2da regla se elimina el padding de la lista ul y se establece un ancho de 100% del espacio para que entren los botones.
  • En la 3ra regla, los renglones que se comportan en bloque por defecto, cambiamos su comportamiento a en línea con display para que entren en el espacio creado anteriormente; con el valor inline.
  • Para los enlaces: Le sacamos el subrayados y el contorno por defecto. Los links se muestran en línea pero queremos que los elementos consecutivos solo ocupen una línea sin separase y tambien queremos dimensionarlos y eso los conseguimos con inline-block.
  • Para la dimension de los enclaces: Llenamos todo ul con botones del mismo tamaño y un margin (espacio) enter ellos, sin hueco en el borde derecho, lo hacemos con width en 19.5% y margin-right con 0.625%. Pero como suma 100.625% y se desborda lo correguimos con una siguiente regla: li:last-child a con margin-right en 0 que toma el ultimo li para eliminar el margen. Para la altura del boton, usamos line-height en 3 con el texto centrado.

En el código del HTML, las lineas de la lista estan juntas es porque los elementos en bloque crean espaciado en la página, como se recomienda para el valor inline-block en CSS-Tricks.
