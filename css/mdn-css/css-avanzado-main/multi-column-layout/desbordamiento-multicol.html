Desbordamiento en diseño de multicolumna

Puede suceder dentro de las columnas y cuando hay más contenido que espacio en el contenedor.

Desbordamiento dentro de las cajas de columnas

Cuando el tamaño del elemento (dentro de columna) es mayor que la columna. Por ejemplo, una imagen es más ancha que una columna. El contenido no se recorta, se desborda

La solución es seguir el estandar para las imagenes, max-width: 100%;.

.container { column-width: 150px; }

Solución:

Más columnas de las que caben en el contenedor

Las formas en que se manejan las columnas desbordadas depende si el documento es fragmentado (pdf, libros, etc) o si es continuo (un sitio web, audio, video).

En medios fragmentados, despues de que se llenen las columnas de una página, las columnas se moveran a una nueva página. En medios continuos, las columnas se desbordarán hacia la dirección en línea (texto/contenido) y el navegador pondrá una barra horizontal.

Ejemplo, el contenedor tiene un ancho definido y hay más texto que espacio para crear columnas; obtenemos columnas creadas fueras del contenedor.

En una futura version de especificación podría haber columnas que aparezcan abajo cuando no haya espacio horizontal.

Uso de media quieries verticales

Se nos presenta el problema de que cuando hay muchisimas columnas, la ventana del navegador no puede abarcar todas; entonces, para leer hay que desplazarse arriba o abajo, en este sentido no es responsivo.

Por ejemplo, podemos verificar la altura de la ventana con un media query, antes de aplicar las propiedades de multicol

@media (min-height: 300px) { .container { column-width: 200px; } }

Cuando tengamos una altura minima de 300px, los contenedores serán de 200px. Así facilitaremos la lectura del contenido.
