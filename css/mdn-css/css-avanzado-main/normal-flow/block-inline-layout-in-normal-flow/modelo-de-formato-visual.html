Modelo de formato visual

Describe como los navegadores leen el DOM (árbol del documento) y lo muestran para los medios visuales (continuous media) una página, audio, video; y medios páginados (paged media) como puede ser un pdf para imprimir, un libro, etc.

Cada elemento del DOM o del documento (que creamos: parrafo, link, etc) que puede ser una página web o pdf se muestran en pantalla por el siguiente orden:

  • Las cajas de CSS y su tamaño
  • Su posicionamiento en normal flow, float y position absolute.
  • Relaciones entre elementos en el árbol del documento.
  • Información externa (tamaño del viewport, tamaño de imagenes).

El rol de viewport

En medios continuos o visuales (por ej, una web) podemos cambiar el tamaño de la ventana del navegador (en escritorio) o rotar la ventana (movil).

Si el documento es muy grande, tiene muchos elementos o mucho contenido; mostrará barras para desplazarse.

Generación de cajas

Es la parte del modelo visual que crea cajas a partir de los elementos que hay en el documento. Hay diferentes tipos de cajas, depende de su valor en la propiedad display.

CSS toma el documento y lo representa en el lienzo, genera una estructura de cajas (intermedia) que es la estructura de formato del documento renderizado (mostrado). Cada caja en el árbol de cajas, representa su elemento correspondiente (o pseudo-elemento) en el espacio/lienzo, mientras que el texto en el árbol, representa el contenido de los nodos de texto.

Luego, para cada elemento, CSS general cero o más cuadros segun el valor de display de cada elemento.

La caja principal

Un elemento genera o contiente otras cajas, puede ser el elemento body.

Si un elemento tiene display: list-item; y valores com none o contents hacen que él o sus elementos descendiente, no generan ninguna caja.

Cajas anonimas

Se crea cuando no hay un elemento de HTML para usar en una caja. Por ejemplo, cuando declaras display flex en un contenedor/elemento principal y hay texto que no está encerrado en un parrafo. El arbol de cajas, creará una caja anonima para contener ese texto. Se comportará como un elemento flexible pero no se puede diseñar como una caja normal porque no podemos llamar al contenido suelto.

Lo mismo puede suceder cuando en un div hay contenido de texto suelto sin el elemento parrfo. Y lo mismo pasa con otros metodos de diseño.

Tambien sucede con las cajas en línea cuando hay uno de estos elementos vacio.

Esquemas de posicionamiento y elemento in-flow y out-of-flow

Un elemenento/caja se puede diseñar de acuerdo con tres esquemas de posicionamiento, normal flow, floatsy position absolute.

Normal flow

El flujo normal incluye las cajas en bloque, en línea y los elementos en bloque o en línea con la propiedad position relative y sticky.

Float

A una caja normal, la podemos transformar en flotante con la propiedad float, se posicionan fuera del flujo normal, pueden estar colocados a la izquierda o derecha de una página.

Posicion absoluta

Un elemento con la propiedad position: absolute; se posiciona fuera del flujo normal y se le asigna un contenedor.

Un elemento está out-of-flow si es un flotante, tiene posicion absoluta o si es el elemento raíz (html). Un elemento está in-flow si no está fuera del flujo porque no es flotante, absoluto o es el elemento raíz.
