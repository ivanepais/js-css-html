Diseño en grid y accesibilidad

El objetivo es crear un documento semántico y bien estructurado y luego aplicar CSS para crear el diseño que deseáramos. CSS Garden nos desafia a hacer esto y crear un diseño único.

Una cuadrícula se define en el CSS y no en el HTML, separá el contenido del marcado. Sin embargo veremos si podemos tener problemas de accesibilidad.

Reordenar el contenido en grid

Con grid podemos darle otra forma al contenido de nuestra página de varias maneras: con la propiedad order que cambia la forma en que los subelementos se colocan automaticamente. Podemos usar grid-auto-flow: dense va acomodando los elementos según entren en el espacio disponible (cambiando el orden DOM). Tambien podmeos usar grid-area para ordenar según las líneas o para crear areas.

La especificación de grid hace referencia a lo que sucede con la accesibilidad cuando se reordenan los elementos: no afecta al orden del Dom ni afecta a los lectores de pantalla ni al orden de navegación con la tecla tab.

Solo se utilizarán estas propiedades para el reordenamiento visual y no para el orden lógico.

Re-orden visual no es el lógico

Cuando reordenamos cosas, sea en flex o grid; solo es visualmente. El código fuente es lo que controla el texto a voz y el orden de tabulación.

Si queremos hacer un reordenamiento lógico lo hacemos desde el código fuente, si queremos hacer un reordenamiento visual lo hacemos desde el CSS.

.box1 { grid-column: 1; grid-row: 2; }

Como debemos afrontar la accesibilidad desde Grid

Debemos priorizar un orden lógico, tambien debemos ver como mantener la accesibilidad para todos los tipos de usuarios cuando interactuan los elementos en la página.

Comenzar con un documento estructurado y accesible

En principio debemos asegurarnos desde el HTML que los elementos dispongan de accesibilidad y mantener un orden lógico en los elementos, sin tener que cambiar el código fuente para lograr el diseño deseado. Beneficia a todo tipo de usuarios.

Crear un diseño responsivo y grid responsivo

Cuando usamos media queries para ir cambiando el diseño a la medida de los diferentes dispositivos debemos priorizar el orden lógico a el orden visual de los elementos, dejando estos para lo ultimo.

Revisando el código fuente

No deberias usar las propiedades de CSS para cambiar el orden lógico de un elemento, para eso deberías cambiardo desde el código fuente.

Grid y el peligro del markup flattening

Se dice aplanar el marcado cuando queremos que elementos no secunadarios de un contenedor se comporten como si fuerán a pertenecer a él.

El valor de subgrid grid-template-columns y rows, resolverá este problema heredando la cuadrícula a estos elementos, pero no está completamente implementado por los navegadores.

Entonces, se crean divs en lugar de usar elementos semanticos ya que es más facil hacer que sean elementos secundarios ya que es una etiqueta independiente; distinto de lo que sería una lista.

Esto se evita marcando los elementos con su etiqueta correspondiente y usando el CSS correctamente sobre los elementos para no tener que usar ese tipo de trucos.
