Box Alignment en Grid

La especificación Box Alignment Level 3 determina las propiedades y reglas para la alineación de cajas en todos los metodos de diseño, siempre que para ellos sea posible; por eso es que vemos pequeñas diferencias. Por otro lado, tambien depende de que los navegadores la apliquen, en el caso de Grid está ampliamente aceptada.

En grid empezaremos entendiendo los dos ejes.

Los dos ejes de la cuadrícula

Dispone de dos ejes para alinear las cosas: el eje en bloque y el eje en línea. El eje en bloque es la dirección hacia donde van apareciendo los elementos, cruzando al eje en línea. El eje en línea es la dirección hacia donde corre el contenido/texto, cruzando al eje en bloque.

Podemos alínear el contenido que hay dentro de un area/celda y cuadrícula entera.

Alinear elemenos en el eje en bloque

Se usa las propiedades align-self y align-items, dentro de las areas o cuadrícula completa. Se usa en el contenedor para la direccion vertical/aparecen los elementos.

Usando align-items

Alineará todos los elementos de la cuadrícula, dispone de los siguientes valores:

  • auto
  • normal
  • start
  • end
  • center
  • stretch
  • baseline
  • first baseline
  • last baseline
grid-template-areas: "a a a a b b b b" "a a a a b b b b" "c c c c d d d d" "c c c c d d d d"; align-items: start; } .item1 { grid-area: a; }

Usando align-self

Podemos alinear uno por uno los elementos o despues de alinear todos, elegimos un elemento necesite una alineación distinta a los demás.

.item4 { grid-area: d; align-self: center; }

Elementos con relación de aspecto intrinseca

El comportamiento predeterminado de align-self es stretch (estirar), excepto para los elementos multimedia que tienen una relación de aspecto con el valor start; es para que no pierda su tamaño.

Los navegadores no lo tienen implementado según la especificacion por eso hay que configurarlo manualmente.

Debemos poner las propiedades align-seft y justify-self con el valor start.

Justificando elementos en el eje en línea

justify-items y justify-self hacen el mismo que las propiedades anteriores pero para el eje en línea. Podemos acomodar todos los elementos de la cuadrícula y un elemento individual dentro del área.

Los valores son los mismo: auto, normal, start, end, center, stretch, baseline, first baseline, last baseline.

el valor predeterminado es stretch, excepto para elementos con una relación de aspecto intrínseca. Debemos aplicar la tecnica anterior, para respetar la especificación y hacer buena practica poniendo justify-self: start para los objetos multimedia como pueden ser las imagenes.

.item2 { grid-area: b; justify-self: start; }

Propiedades abreviadas de alineación

place-items es la abreviada para align-items y justify-items para todos los elementos en los dos ejes. Se aplica en el contenedor.

place-self actua de la misma forma. ES para los elementos individuales, se aplica en ellos, no en el contenedor.

Centrar elemento en un área

Si combinamos align-self y justify-self en un elemento lo podemos centrar en un area/cuadrícula entera.

grid-template-areas: ". a a ." ". a a .";

Alinear tracks (filas y columnas) en el eje de bloque

Cuando tenemos una situación en la que las pistas de la cuadrícula utilizan un área que es más pequeña que el contenedor de la cuadrícula (sobre todo en espacios con tamaño fijo), entonces puede alinear las pistas de la cuadrícula, dentro de ese contenedor.

Esto funciona para los ejes de bloque y de línea. Usamos la propiedad align-content para el eje de bloque y justify-content para el eje de línea.

Tambien hay una propiedad abreviada, place-content que junta las dos propiedades, las propiedades se utilizarán en el contenedor. los valores para todas son:

  • normal
  • start
  • end
  • center
  • stretch
  • space-around
  • space-between
  • space-evenly
  • baseline
  • first baseline
  • last baseline

Por ejemplo, si tenemos una cuadrícual de 500x500 y hay columnas y filas (areas) menor a este valor, hay espacio para usar las propiedades de alineación.

La propiedad align-content funciona en el contenedor y mueve todas las pistas en la direccion del eje de bloque.

Comportamiento por defecto de las filas y columnas

Tienen el valor start de la propiedad grid-area usando sus líneas, empiezan en la esquina superior izquierda.

Align content end en el eje en bloque

Si lo usamos en el contenedor, desde la posición por defecto start, moverá todas las filas y columnas hacia abajo.

Otros valores de align-content

Con sus valores movemos todos los elementos/pistas en el eje de bloque, recordando que la condición es que siempre que haya espacio.

Si usamos space-between los elementos tomarán más espacio.

Los valores space-between, space-around y space-evely afecta a todos los elementos distribuyen el espacio y hacen que los elemenentos sean más grandes que abarcan más pistas, como es una cuadrícula en de tamaño fijo, debemos asegurarnos que el contenido pueda entrar sin desbordarse.

justify-content en para el eje en línea

Si usamos el valor space-around separa los elementos en todas las líneas.

Alineación y auto margen

Al configurar el margen izquiedo y derecho en auto con margenes verticales en 0, centramos centramos los elementos de bloque. El valor auto absorve el espacio utilizado.

.item1 { grid-area: a; margin-left: auto; }

En el ejemplo tenemos, al elemento 1 le damos margin-left: auto que empuja todo el contenido hacia el otro lado por el espacio que no se haya utilizado, despues de que de demos espacio en el area.

Podemos usar las herramienentas para desarrolladores de grid para resaltar las lineas, las filas y columnas para ver estos espacios creados.

Alineación y modos de escritura

Grid y las propiedades de alineación estan pensadas para soportar todos los modos de escritura, significa que los bloques y el contenido fluye de otra manera.

En el modo de escritura horizontal-tb que es el que se usa en ingles, las lineas de inicio de los elementos estan arriba y a la izquierda.

Pero si usamos márgenes automáticos, margin-right o margin-left, o position absolute a los elementos usando los desplazamientos superior, derecho, inferior e izquierdo no respetaría los modos de escritura.

Se deben usar las propiedaes relativas al flujo (de los bloques y el contenido) en lugar de las propiedades fisicas, como lo son las anteriores. Es importante tenerlo en cuenta para desarrollar sitios en varios idiomas o para mostrar varios idiomas (se usa html) o para diseñar usando diferentes writting-mode (cambio estetico/artistico a los elmentos y no para idiomas).
