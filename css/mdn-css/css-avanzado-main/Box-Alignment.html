CSS Box Alignment

Especifica la alineación en los distintos metodos de diseño: block, table, flex y grid. En cada metodo cambiaran algunas cosas.

Alineación heredada

Se denomina heradado a las formas de hacer algunas cosas antiguamente, en lo que respecta a la alineación de cajas: para alinear texto usabamos text-align, centrar bloques usando margin: auto a la izquierda y derecha de cada caja y para los diseños de tabla o bloques en línea se usaba vertical-align.

Ejemplos basicos

Tanto en grid como flexbox, justify-content se usa en el eje principal y align-items en el eje secundario o transversal.

Para grid, el eje principal es la dirección en línea (texto) y el eje secundario es la dirección en bloque (cajas).

En flex, el eje principal se alinea con justify-content y el transversal se alinea con align-items. Con la propiedad flex-dirección podemos cambiar la dirección de los ejes, sin necesidad de cambiar el writing-mode.

Los ejes dependen del writing-mode y las propiedades de alineación se aplican en el contenedor.

Ejemplo en Grid

Podemos sobreescribir la alineación del contenedor para algún elemento.

.box { display: grid; width: 540px; grid-template-columns: 120px 120px 120px; align-items: start; justify-content: space-between; } .box :first-child { align-self: center; }

Ejemplo en Flexbox

Podemos hacer lo mismo del ejemplo anterior.

.box { display: flex; align-items: flex-start; justify-content: space-between; } .box :first-child { align-self: center; }

Conceptos claves y terminología

La especificación determina conceptos para facilitar la comprensión del apartado Box Alignment.

Relación con los modos de escritura

La alineación está vinculada a los modos de escritura, no depende de los lugares fisicos de la caja como arriba, abajo, etc. Si no desde dónde empieza y termina un elemento, esto se ve facilmente en los valores de writing mode; de manera que la alineación funcione sea cual éste valor.

Las dos dimensiones de alineación

las propiedades de alineación de cuadro, alineará el contenido en uno de dos ejes: el eje en línea/principal (hacia donde fluyen las palabras) y el eje de bloque/transversal (hacia donde fluyen las cajas/elementos, atraviesa al primero).

Para alinear en línea se utilizan las propiedades justify-content/items/self. Para los elementos en bloque se utilizan las propieades align-content/items/self.

En flexbox, si cambiamos el valor de flex-direction: row a column, se invierte la dirección de las palabras y los elementos. Aquí es más facil pensar en eje principal (justify) y en eje transveral (align) esto es lo que no se cambia.

Sujeto y contenedor de la alineación

Es lo que se está alineando, en justify-items o align-items es la caja de los elementos. En justify-content o align-content es el contenedor de elementos.

Fallback en la alineación

Si establece una alineación que no se puede cumplir, la alineación alternativa entrará en juego y se ocupará del espacio disponible. Esta alineación alternativa se especifica individualmente para cada método de diseño.

Tipos de alineación

Hay tre tipos de alineación, son valores en palabras claves:

  • Positional alignment: es la posición de un sujeto de alíneación en relación con su contenedor.
  • Baseline alignment: está palabra clave, define la alineación como una relación entre las líneas de multiples sujetos de alineación dentro de un contexto de alineación.
  • Distrituted alignment: sus palabras claves definen la alineación como una distribución de espacios entre los sujetos/elementos/cajas de alineación.

Palabras claves de Positional alignment

Son valores para justify-content/items/self y align-content/items/self.

  • center
  • start
  • end
  • self-start
  • self-end
  • flex-start (solo para flexbox)
  • flex-end (solo para flexbox)
  • left
  • right

Los valores fisicos como left y right se relacionan con los lados de la pantalla/caja pero los valores lógicos se relacionan con el inicio y final de los elementos en el writing-mode.

Por ejemplo, en Grid si ponemos justify-content: start es una propiedad para el eje en línea/principal. Si estamos en modo de escritura ingles, el inicio de los elementos están a la izquierda de la pantalla.

En arabe, empiezan desde la izquierda (start) hacia la derecha (end).

Alineación para Baseline alignment

Los valores se usan para alinear las líneas de base a traves de un grupo de sujetos o elementos. Funcionan con las propiedades justify-content o align-content.

  • baseline
  • first-baseline
  • last-baseline

Funcionan en las líneas de los metodos de diseño, los elementos/sujeto se alinearán en la línea base agregando paddig o relleno en éstas.

Self alignment en baseline mueve las cajas/elementos para alinearlos por línea de base agregando un margen fuera de los cuadros. Cuenta como self alignment cuando se usa justify-self o align-content, o cuando se establecen estos valores en un grupo de elementos con justify-items o align-items.

Distributed alignment

Sus valores definen lo que sucede con cualquier espacio adicional después de que se hayan mostrado/puesto los elementos en el sitio. Se usan en justify-content y align-content, sus valores son:

  • stretch
  • space-between
  • space-around
  • space-evely

Debe haber espacio disponible en la dimensión en la que desea alinear los elementos para que estas palabras clave surtan efecto. Sin espacio, no hay nada que distribuir.

Se usan frecuentemente con flexbox para alinear elemenentos dentro de contenedores.

Espacios entre cajas

La especificación para la alineación tambien incluye las propiedades column-gap, row-gap y gap para los espacios entre los elementos. Permiten los espacios fijos entre filas columnas, elementos, etc.

Desbordamiento en la alineación

Tiene palabras claves como unsafe y safe que determinan un comportamiento cuando los elementos alineados son más grandes que el contenedor. Safe se alineará el elemento al inicio en caso de que se provoque un desbordamiento, para que no se pierda contenido. Unsafe dejará el desbordamiento, incluso si hay perdida de contenido.
