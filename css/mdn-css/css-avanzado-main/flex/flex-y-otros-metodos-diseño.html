Modulo de alineación de cajas

Algunas propiedades de alineación de elementos flexibles son parte del Modulo de alineación de cajas que especifica las propiedades para la alineación de elementos en todos los metodos de diseño: normal flow, flex, grid, etc.

Hay una nota en la especificación de flexbox que indica que en el futuro, una vez que se complete, las definiciones en Box Alignment Level 3 reemplazarán a las de flexbox

Writting mode

Flexbox se adapta a los modos de escritura que determina la propiedad writing-mode con sus valores:

  • horizontal-tb: Por defecto, usado en los idiomas en latin.
  • vertical-rl: usado en idiomas y cultura japonesa, china.
  • vertical-lr
  • sideways-rl: (solo en firefox).
  • sideways-lr: (solo en firefox).

Esta propiedad se usa para cuestiones creativas de diseño ya que estamos en CSSS, no se usa para cambiar el idioma de un documento completo. Esto se hace desde HTML con los atributos dir y lang para indicar la dirección de texto y su idioma.

Flex y Grid

Con flex y grid podemos sobreescribir otros metodos, los reemplazamos, no se aplican todos juntos. Si un elemento flexible se convierte en un elemento de cuadrícula, se ignorarán las propiedades flexibles que se hayan asignado a los elementos secundarios. Se puede usar flex para reemplazar un grid, en un navegador antiguo para dar compatibilidad y no cambiar tanto el diseño, ademas podemos usar las propiedades de Box alignment para alinear los elementos.

Diferencias:

  • Diseño unidimensional en flex (todos los elementos se agrupan en fila o columna) versus, diseño bidimensional en grid (los elementos se agruparán en filas y columnas).
  • Hay diferencias en cuanto a como se ordenan (en flex se ubican todos los elementos seguidos, en una dirección; en grid se puede elegir el orden de un elemento, le damos una ubicación en la cuadrícula), como se configuran los espacios entre los elementos, etc.

    • En algunos casos se pueden usar los dos, pero en otros es mejor uno u otro. Aunque con el tiempo se dará cuenta cual funciona mejor para cada caso.

Flex y la propiedad display: contents

La propiedad display: contents.

Ejemplo, tengo un contenedor flexible con tres elementos secundarios. Uno de estos elementos flexibles tiene dos elementos anidados, que normalmente no participarían en el diseño flexible. El diseño flexible solo se aplica a los hijos directos de un contenedor flexible.

Al agregar display: contents al envoltorio alrededor de los elementos anidados, puede ver que el elemento ha desaparecido del diseño, lo que permite que los dos subelementos se presenten como si fueran elementos secundarios directos del contenedor flexible. Puede intentar eliminar la pantalla: línea de contenido para verla regresar.

Tenga en cuenta que esto solo elimina el cuadro del diseño; los subhijos no se vuelven hijos directos de ninguna otra manera. Puede ver que, como he usado un selector secundario directo para agregar el fondo y los bordes a los elementos flexibles, esto no se ha aplicado a nuestros elementos secundarios anidados. Se han presentado como artículos flexibles, pero como no son hijos directos, no obtienen el otro estilo.

Al haber eliminado estilo con display: contents; no puede usarlo para, por ejemplo, agregar un color de fondo detrás de los subhijos anidados

Esta característica será muy útil en circunstancias en las que necesite el marcado por razones semánticas pero no desee mostrar el cuadro que generaría de forma predeterminada.
