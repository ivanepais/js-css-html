Capitulo 2: Bloques de construcción

Cascada y herencia

Controlan como se aplica el CSS al HTML

Reglas conflictivas:

Por lo general, son varias reglas aplicadas a un mismo elemento; hacen que el resultado no sea el esperado.

La cascada y la especificidad controlan que regla se aplica en este conflicto. La herencia significa que algunas propiedades del elemento padre se heredan (a los elementos hijos) pero otras no.

Cascada

Cuando dos reglas tienen la misma especificidad, se aplica la que aparece ultima: Es decir, tener un selector idéntico

Especificidad

Cuando tenemos selectores diferentes que podrían aplicarse a un mismo elemento. Si un elemento es mas especifico que otro se aplicará este a pesar que este antes en la hoja de estilo.

Herencia (inheritance)

A los elementos que estén dentro de el se le aplicaran las mismas propiedades, pero a otros no. No se heredara propiedades si el elemento hijo tiene las suyas.

Los bordes, los margenes, el relleno y la anchara no se heredan en una ul, por sentido comun.

Ver Referencia de cada propiedad para ver cuales son heredables

Control de la herencia: Valores

inherit

Es un valor permitido en todas las propiedades CSS. Hace que el elemento al cual se aplica tome el valor calculado de la propiedad de su elemento padre.

Si hay un elemento hijo con la misma especificidad se aplicará las que tenga definida y en orden a la cascada.

initial

ES un valor que se puede aplicar a un elemento seleccionado tenga el mismo valor que esté establecido para esa propiedad en la hoja de estilo por defecto del navegador. Fuerza la utilización del valor inicial de la propiedad para el elemento seleccionado.

unset

Este valor restablece la propiedad a su condición natural, si no es heredable, sera inicial

revert

Este valor actuá como el valor unset (restablece valor natural) en muchos casos. También puede establecer los valores de estilo del navegador.

all

Es una propiedad que puede tomar los valores anteriores para restablecer el estilo que tenia el elemento antes de hacer cambios (funciona para casi todos las propiedades de estilo que se tenían aplicados).

Comprender la cascada:

Veremos la forma en que la cascada define que reglas se aplican según estos tres factores: orden del código, especificidad e importancia.

La importancia invalida a la especificidad y esta al orden del código de CSS.

Orden del código

Si hay mas de una regla con el mismo peso/especificidad gana la ultima.

Especificidad

Cuando se aplica una regla anterior es porque esta es mas especifica que la ultima.

No se sobrescribe toda la reglas, solo las propiedades que entran en conflicto de especificidad/valor. Una practica común es dar estilo a elementos básicos y después crear clases para cambiar otras propiedades del elemento. El estilo genérico se aplica a todos los elementos y el de las clases solo a elementos particulares, estos van a recibir las propiedades que no se hayan sobrescrito con mayor especificidad.

Ejemplo: A un selector genérico le ponemos color rojo y fuente Helvetica y a un selector de clase le ponemos color azul sin cambiar la fuente, este recibirá la fuente Helvetica pero cambiará el color azul.

Calculo de la especificidad

Un selector genérico o pseudo elemento suma 1 punto, un selector de clase suma 10 puntos, un selector de id suma 100, un atributo style en HTML suma de 1000 puntos; pero la declaración !importan al lado de un valor le gana a todos los selectores.

declaración !important

La unica manera de anularla es escribiendo otro !important al valor de la propiedad que se quiere aplicar.

Aunque esta declaración no se utiliza a menos que no puedas acceder a modificar los nodos (elementos, texto, etc.) del CSS.

Efecto de la ubicación del CSS

Declaraciones contradictorias se aplicaran en el siguiente orden:

  1. Declaraciones en las hojas de estilo del cliente/navegador
  2. Declaraciones en las hojas de estilo del usuario web
  3. Declaraciones en las hojas de estilo del desarrollador web
  4. Declaraciones !important en las hojas de estilo del desarrollador
  5. Declaraciones !important en las hojas de estilo del usuario web

Cascada en detalle

La cascada es un algoritmo que define como combinar los valores de las propiedades en las hojas de estilo. Como una cascada, se aplicará al documento (html, xml, etc) de la ultima hacia la primera regla en general, aunque hay casos particulares.

Solo las reglas con par propiedad/valor participan de la cascada, significa que las reglas de CSS distintas o que respondan a otro algoritmo que no sea el de CSS no participaran.

Hay declaraciones como pueden ser algunas de las reglas (at-rules) como @font-face que contiene descriptores. Declaraciones mas conocidas como @media, @document o @supports; si participarán de la cascada.

At-rules

Es una declaración que comienza con el signo @ seguido del nombre de la regla.

Hay reglas con distintas sintaxis y reglas con una misma sintaxis pero que funcionan mediante condiciones.

Orden de la cascada

La cascada determina como va a encontrar el valor que se aplicará a cada propiedad de cada elemento.

Primero clasifica las reglas que se permiten en la cascada (con par propiedad/valor y que responden al algoritmo de la cascada de CSS), luego clasifica las reglas de acuerdo a su importancia y luego por su origen (tipo de hoja de estilo: cliente, autor y usuario), estas hojas determinan la prioridad: 3) cliente, 2) autor; 1) usuario. El autor puede modificar la hoja del cliente y el usuario puede modificar la hoja del autor.

Cómo elige el navegador que estilo aplicar, en orden:

  1. cliente/browser
  2. usuario
  3. autor
  4. animation
  5. autor (!important)
  6. usuario (!important)
  7. cliente (!important)
  8. transición

Finalmente, si hay igualdad entre las reglas, la especificidad elige el orden en el cual se aplicará el CSS.

Aunque lo más recomendable es no romper la cascada al aumentar tanto la especificidad, mantenerlas igualadas para que la ultima reglas se siga aplicando.
