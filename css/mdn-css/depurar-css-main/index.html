Capitulo 10: Depurar CSS

A veces, al escribir CSS no hace lo que esperas. Aquí es cuando usamos las Dev Tools de los navegadores para ver qué está pasando por detras de la web y solucionar los problemas.

Acceder a las DevTools de los navegadores

Generalmente en los navegadores más utilizados, se abre con f12 o ir al menu - herramientas - herramientas del desarrollador o haciendo click derecho a un elemento de la web e inspeccionar el elemento. Veras que hay varios apartados que siren ver y hacer diferentes cosas, explicadas más adelante.

Es necesario hacer pruebas en los navegadores más utilizados del mercado, aunque tambien hay navegadores que tienen herramientas especializadas en algun area.

Uso y Herramientas del desarollador - Basico

Inspector: Explorador del DOM y editor de CSS

Se ve el HTML y el CSS que se está aplicando hasta lo podemos modificar y ver el resultado instantaneamente.

Explorando el DOM

Hacemos click derecho en un elemento del HTML y podemos ver distintas opciones:

  • Eliminar nodo/elemento: Elimina el elemento en el que estamos posicionados.
  • Editar como HTML/Agregar atributo/Editar texto: Permite cambiar el HTML y ver resultados. Sirve para depurar y probar.
  • Copiar/Copiar como HTML: Copia el HTML seleccionado.
  • Copiar ruta CSS y XPath: Para copiar el selector CSS o XPath que selecciona el elemento HTML posicionado
  • Estados de los elementos: Si un elemento seleccionado los tiene, lo podemos activar para verlos.

Podemos realizar los cambios que deseemos pero no los podemos guardar.

Editor de CSS

Cuando abrimos las DevTools o seleccionamos un elemento, en la segunda columna nos encontramos con el CSS aplicado a la web. Con algunos apartados:

  • Las reglas aplicadas al elemento selecionado se muestran de mayor a menor especificidad.
  • Casillas de verificacion de las reglas: si la desactivamos en algun elemento, este cambia su aspecto por uno predeterminado.
  • Al lado de las propiedades (abreviadas) hay una flechita que si la activas te muestra las propiedades individuales aplicadas a la página.
  • Si una propiedad y valor esta disponible, tocando su nombre o valor lo podemos cambiar. Si no está disponible podemos crear una regla.
  • Junto a cada regla está el nombre del archivo y el numero de línea para poder editar y guardar. Lo vemos arriba a la izquierda.
  • Si hacemos click en la llave de cierre de la declaración, nos abre un cuadro en donde podemos escribir una nueva regla completa.

En la tercera columna notaras algunas pestañas que proporciona otro tipo de info:

  • Calculado: Muestra el estilo calculado para el elemento seleccionado
  • Diseño/layout/distribución: incluye secciones relacionadas con la visualizacion de la página como:
    • Box-model: Muestra el modelo de caja para el elemento seleccionado y sus valores
    • Cuadricula/Grid y Flexbox: Si utilizamos alguno de estos metodos de diseño, se mostrará sus detalles.
  • Fuentes: Muestra los tipos de letra aplicados al elemento.

Puedes ver más info en la documentacion de los navegadores con respecto a su DevTools.

Consola y Depurador de JS

Al lado de la pestaña de inspector podemos ver estas dos opciones que se utilizan para JavaScript, la utilizaremos en su curso.

Depurando CSS con Firefox

Primero abrimos el documento que vamos a utilizar para practicar.

Cuando miramos el codigo fuente con el click derecho - ver codigo fuente nos aparece el HTML que escribimos nosotros en una pestaña nueva. Es distinto cuando miramos el codigo desde la DevTools del navegador, este va a mostrar un correcciones si escribimos mal el codigo

Esto lo hace el DOM y lo que vemos es el arbol de HTML que nos muestra lo que representa para el navegador el codigo que escribimos.

Inspeccion del CSS

Seleccionamos cualquier elemento con la DevTools, por ejemplo, la clase box1; primero vemos el arbol de HTML y al lado vemos el panel de CSS que tiene las reglas aplicadas a este elemento y tambien el CSS heredado de su elemento padre body. Es util cuando se aplica un CSS que no esperabas porque se esta heredando de un elemento padre y necesitas sobreescribirlo con una nueva regla para este elemento.

En este mismo apartado del panel, podemos expandir las propiedades abreviadas y se mostraran las individuales con la flechita izquierda.

Tambien podemos deshacer reglas si pasamos el mouse sobre las declaraciones, nos aparece una casilla que cuando la desactivamos quitará este CSS. Podemos usarlo para hacer una comparacion A y B para elegir cual regla es mejor. O si un diseño falla podemos ver qué causa el problema.

Editar valores

Es util para ver diferentes tamaños, colores, podemos ahorra mucho tiempo al editar una hoja de estilo y volver a cargar la página.

Seleccionamos box1 y en border vemos que hay un color, en el circulo clickeamos y nos aparecerá un selector para probar en tiempo real distintas opaciones. Como tambien podemos cambiar el ancho o el estilo de este borde.

Añadir propiedad nueva

Por ejemplo, si queremos que la caja no herede el tamaño de letra del body; podemos añadir una nueva declaracion en el elemento. Hacemos click en la llave de cierre para agregar la propiedad.

Arriba de las reglas hay un boton + que permite agregar reglas en el selector posicionado o usando otro.

En este panel tambien podemos ver reglas tachadas porque sus valores no estan permitidos.

Entendiendo el modelo de cajas

Tenemos el Box Model y el modelo alternativo, en la tercera columna con la pestaña layout/diseño/distribucion vemos el model box de los elementos seleccionados. Incluye propiedades declaradas y las que vienen predeterminadas

Abajo del diagrama se detallas las propiedades del modelo. Una muy importante es box-sizing que controla el modelo de cajas que usa el elemento.

Box1 usa content-box que toma el tamaño que le asignamos al elemento y luego añade el padding y el ancho de border. O sea, al tamaño inicial se le suman los demas y el elemento queda más grande.

Box2 usa border-box, el tamaño de toda la caja va a ser el ancho y alto que le asignemos, si usamos padding y border se va a restar ancho y alto pero va a quedar de igual en tamaño total.

Resolver problemas de especificidad

Cuando necesitamos editar el estilo de una pagina ya publicada y nos resulta dificil que se aplique un estilo, lo que suelo suceder es que un selector más especifico anula los cambios

En nuestro ejemplo vemos el elemento em (cursiva) en colores distintos. En la hoja de estilo primero esta el selector de clase ".special con un color naranja y despues aparece un selector de etiquea em de color rosa. El primer selector es más especifico que el segundo, no importa si está escrito antes. En DevTools vemos que el selector de clase ".special" anula al selector de elemento.

Problemas al depurar CSS

Cuando no podemos resolver los problemas con las herramientas de desarrollador, los siguientes apartados te pueden ayudar.

Alégate del problema

Especialmente en los problemas con CSS porque no recibimos mensaje de error ni dónde está el problema, ni la documentación para solucionarlo -como sucede en JS-. Entonces, cuando nos sentimos agobiados, frustrados de tanto intentar y no encontramos la solución es mejor alejarnos del problema, salimos a algun lugar, hablamos con un compañero, la solucion puede llegar cuando nos sentimos más frescos.

El HTML y el CSS son validos?

Los navegadores harán todo lo posible para mostrar nuesta página incluso con errores, por lo tanto, un primer paso es pasar el docuemnto HTML y CSS por un validaddor:

  • Validador de CSS
  • Validador de HTML

La propiedad y el valor son compatibles con el navegador?

Los navegadores ignoran lo que no entienden, si la propiedad o valor no es compantible con el navegador, el CSS no se aplicará. Las DevTools tienen un icono triangular de precausion en la pestaña de las reglas y declaraciones de CSS

Tambien podemos mirar las tablas de compatibilidad que cada propiedad tiene en la documentación.

Hay algo más que anule el CSS?

Aquí es donde el conocimiento sobre especificidad, cascada y herencia es de gran ayuda.

  • Si tenemos algo mas especifico que lo que intentamos hacer, usamos la DevTools para ver que estilo se ha aplicado verdaderamente. Así podemos hacer un selector bien especifico para lo que queremos hacer.

Haz un caso de prueba reducido del problema

Es una habilidad muy util, ayudará a encontrar problemas en el codigo para informar errores o pedir ayuda adecuadamente al identificar los elementoss en conflicto.

El metodo es mostrar el problema en codigo reducido lo más simple posible sin contenido y estilo complejo. Sacar el codigo problematico del diseño para que solo muestre este codigo o caracteristica.

Para crear un caso de prueba reducido

  1. Si el marcado se genera dinamicamente, por ejemplo, desde un CMS creamos una version estatica de la salida que muestre el problema. CodePen
    2. podemos alojar los casos de prueba, compartirlos. Copiamos el todo el HTML del codigo fuente, CSS y JS relevante, podemos verificar si el problema sigue ahí.
  2. Si eliminar JS no soluciona el problema, no lo incluyas. Si hace desaparecer el problema, elimina la mayor cantidad de JS que puedas; deja solo las causas del problema.
  3. Elimina cualquier HTML que no contribuya al problema. Elimina componentes o elementos principales del diseño. Intenta reducir al minimo la cantidad de codigo que aun muestra el problema.
  4. Elimina cualquier CSS que no afecte al problema.

En el proceso, puedes descubrir que causa el problema o al menos ser capaz de ponerlo o quitarlo al eliminar algo especifico. Podemos escribir comentarios en el codigo a medida que se descubre cosas. Si pedimos ayuda, s comentarios indicaran lo que hemos intentado. Esto proporciona info que permite buscar soluciones alternativas

Aun sino da resultado aislar el codigo, podemos acudir a un foro o mostrarlo a un compañero. Puedes describir lo que has intentado con el codigo reducido, un desarrollador más experimentado podrá orientarte.

En caso de que el problema sea del navegador, podemos presentar el codigo aislado como un informe de error al proveedor del navegador en su sitio oficial de bugs.
